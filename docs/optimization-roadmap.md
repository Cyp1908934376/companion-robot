# 架构优化清单

## 1. 电源管理（缺失严重）

当前只提了电池容量，没有电源策略。小机器人续航是核心痛点。

### 1.1 功耗预算 (Phase 1, 3.7V 2000mAh)

```
模块              典型功耗    峰值功耗
──────────────────────────────────────
ESP32-S3 (WiFi)   240mA      350mA
ESP32-S3 (BLE)    80mA       120mA
ESP32-S3 (Light Sleep)  0.8mA      -
摄像头 OV2640     40mA       60mA
麦克风 × 4        10mA       15mA
IMU BMI270        1mA        3mA
环境传感器         2mA        5mA
ToF × 3           15mA       30mA
OLED屏            20mA       40mA
LED环 × 8         10mA       480mA (全白)
N20电机 × 4       200mA      800mA (堵转)
舵机 × 2          200mA      400mA
扬声器            50mA       300mA
──────────────────────────────────────
活跃总计:         ~800mA     ~2.5A
待机(仅WiFi):     ~300mA
深度睡眠:         ~10μA
```

### 1.2 电源状态机

```
┌─────────┐  超时5min无任务  ┌──────────┐  超时30min  ┌──────────┐
│  ACTIVE  │ ──────────────► │   IDLE   │ ──────────► │  LIGHT   │
│ 活跃     │ ◄────────────── │ 空闲     │ ◄────────── │  SLEEP   │
└─────────┘   收到指令       └──────────┘   收到指令   └──────────┘
     │                             │                       │
     │ 电量<10%                    │ 电量<5%               │ 电量<3%
     ▼                             ▼                       ▼
┌──────────┐                ┌──────────┐           ┌──────────┐
│ LOW_POWER│                │ DEEP     │           │ SHUTDOWN │
│ 低功耗   │                │ SLEEP    │           │ 关机     │
└──────────┘                └──────────┘           └──────────┘

ACTIVE:   全功能运行
IDLE:     关闭摄像头, 降低CPU频率(80MHz), 保持WiFi连接
LIGHT_SLEEP: 保持WiFi连接(省电模式), 定时唤醒心跳
DEEP_SLEEP:  仅BLE保持, RTC定时唤醒, 2s唤醒一次检查
SHUTDOWN:    断电, 仅RTC运行, 按键唤醒
```

### 1.3 唤醒源

```
ACTIVE唤醒:
  - 主脑指令 (WiFi/BLE)
  - 唤醒词检测 (低功耗VAD协处理器)
  - 触摸事件 (RTC GPIO中断)
  - 运动检测 (PIR或低功耗ToF)
  - 定时器 (RTC)

LIGHT_SLEEP唤醒:
  - WiFi DTIM信标
  - RTC定时器 (心跳周期)
  - GPIO中断 (触摸/按键)

DEEP_SLEEP唤醒:
  - RTC定时器 (60s检查一次)
  - GPIO中断 (按键)
  - ULP协处理器 (超低功耗, 做简单传感器采样)
```

### 1.4 充电管理

```
充电方案:
  - USB-C 充电 (TP4056, 500mA)
  - 桌面充电座 (弹簧针触点, 1A快充)
  - 电量计: MAX17048 (I2C, 精度±5%)

充电策略:
  - 充电时自动进入IDLE模式
  - 电量>80% 恢复ACTIVE
  - 低电量自动回充电座 (如果底盘支持自主充电)

自主充电 (Phase 2+):
  - 充电座发射红外信标
  - 机器人ToF+IMU导航到充电座
  - 对准后自动对接充电
```

## 2. 定位与导航（完全缺失）

小机器人在室内需要知道自己在哪。

### 2.1 Phase 1: 简易定位

```
方案: 航位推算 (Dead Reckoning) + ToF墙壁参考

传感器:
  - 电机编码器 (N20带编码器版本, +5元/个)
  - IMU (BMI270, 已有)
  - ToF × 3 (已有, 可测墙壁距离)

算法:
  1. 编码器测轮速 → 推算位移
  2. IMU陀螺仪测转向 → 修正航向
  3. ToF测墙壁距离 → 修正累积误差
  4. 简单卡尔曼融合

精度: ~10cm (5m距离内)
局限: 需要知道房间布局, 长距离漂移
成本: +20元 (4个编码器电机)
```

### 2.2 Phase 2: 视觉辅助

```
方案: 视觉里程计 (Visual Odometry)

  - OV2640 低帧率(5fps)特征点追踪
  - 与编码器+IMU融合
  - 精度提升到 ~3cm

  可选: AprilTag标记 (已知位置的二维码)
  - 摄像头检测AprilTag → 精确位姿
  - 精度 ~1cm (在标记附近)
```

### 2.3 Phase 3: SLAM

```
方案: 2D SLAM (RPLIDAR A1)

  - 激光雷达建图
  - ROS2 SLAM Toolbox
  - 主脑运行SLAM算法, 机器人只传激光数据
  - 精度 ~2cm
```

## 3. 多机协同（只提了调度，没提避碰）

50台机器人同场景运行，机器人之间会撞。

### 3.1 集群避碰

```
方案: 主脑集中式避碰

  1. 每个机器人周期上报位置 (心跳附带)
  2. 主脑维护全局位置地图
  3. 调度路径规划时考虑其他机器人位置
  4. 预测碰撞 → 下发减速/停车指令

  问题: 依赖网络, 延迟高时可能来不及

改进: 本地+集中混合

  1. 主脑下发路径规划 (粗粒度)
  2. 机器人本地ToF实时避障 (细粒度)
  3. 机器人间BLE广播位置 (50ms周期)
     → 近距离机器人直接互相避让

BLE位置广播:
  - BLE广播包: [短ID(2B), X(2B), Y(2B), 航向(2B)]
  - 广播周期: 50ms
  - 接收范围: ~10m
  - 检测到近距(<50cm)且路径交叉 → 本地停车
```

### 3.2 编队控制

```
Phase 1: 简单跟随
  - 一个领头机器人 + 其他跟随
  - 跟随者保持固定距离和角度
  - 基于位置反馈的PID控制

Phase 2: 虚拟结构
  - 定义队形模板 (V字/直线/圆形)
  - 每个机器人分配队形中的相对位置
  - 主脑计算并下发目标坐标

Phase 3: 行为协同
  - 基于人工势场法
  - 目标吸引 + 障碍排斥 + 队友排斥
  - 自然形成队形, 无需精确位置
```

## 4. 离线能力（当前架构完全依赖主脑）

网络断了机器人就傻了，需要本地兜底。

### 4.1 离线行为包

```
方案: 主脑预下发行为包, 机器人本地缓存

行为包内容:
  - 基础行为: 巡逻路线、充电回家、避障反应
  - 简单对话: 预录制的问答对 (100-200条)
  - 情绪状态机: 开心/无聊/困倦/好奇
  - 时间触发: 早上问好、晚上提醒休息

存储:
  - NVS分区: 行为配置 (JSON, ~10KB)
  - Flash分区: 预录制音频 (~1MB)
  - PSRAM: 运行时行为状态

下发时机:
  - 注册后主脑自动下发基础行为包
  - OTA更新行为包
  - 运行时动态下发 (如新增对话)
```

### 4.2 离线对话

```
方案: 关键词匹配 + 本地小模型

  1. 唤醒词触发
  2. 语音转文字: 本地小模型 (ESP-SR, 词汇量有限)
  3. 关键词匹配: 简单规则引擎
     "几点了" → 读RTC时间
     "电量" → 读电池电量
     "跳舞" → 执行跳舞行为
  4. 无匹配 → 固定回复 "我在想..." + 上报主脑

局限: 词汇量~200, 不支持连续对话
```

## 5. 开发调试工具（完全缺失）

固件开发没有调试工具会很痛苦。

### 5.1 日志系统

```
ESP-IDF自带ESP_LOG, 但需要远程查看:

方案: 日志上报到主脑

  分级:
    ERROR   → 立即上报 (BCP EVENT指令)
    WARN    → 附带在心跳中
    INFO    → 本地串口 + 可选上报
    DEBUG   → 仅本地串口

  远程日志查看:
    主脑 API: GET /api/v1/robots/{id}/logs?level=warn&limit=100
    实时日志: WebSocket订阅 robot.{id}.log
```

### 5.2 远程诊断

```
主脑可下发诊断指令:

  DIAG_PING:      延迟测量
  DIAG_MEM:       内存使用报告
  DIAG_TASK:      FreeRTOS任务状态
  DIAG_SENSOR:    传感器自检
  DIAG_MOTOR:     电机测试 (低速转动)
  DIAG_AUDIO:     录音回放测试
  DIAG_CAMERA:    拍照上传
  DIAG_REBOOT:    远程重启
  DIAG_COREDUMP:  崩溃转储上传

诊断结果通过 BCP EVENT 上报, 主脑存储并展示
```

### 5.3 模拟器

```
开发阶段没有实体机器人时:

方案: ESP32软件模拟器 (PC端)

  - 用Rust/Python模拟ESP32固件行为
  - 实现BCP协议栈
  - 模拟传感器数据 (随机/脚本)
  - 模拟电机响应 (位置计算)
  - 连接到真实主脑进行集成测试

  好处:
    - 不需要硬件即可开发主脑
    - 可以模拟50+机器人压力测试
    - CI/CD自动化测试
```

## 6. 数据管线（只提了存储，没提怎么用）

传感器数据存了但没说怎么用。

### 6.1 数据生命周期

```
实时数据 (Redis):
  - 机器人在线状态、位置、电量
  - TTL: 心跳超时×3 (15s)

短期数据 (TimescaleDB):
  - 传感器时序数据、指令历史
  - 保留: 7天 (可配置)
  - 7天后自动降采样 (1min平均) 保留30天

长期数据 (PostgreSQL):
  - 设备信息、任务记录、告警日志
  - 保留: 永久

图像/音频 (MinIO/S3):
  - 快照、录音
  - 保留: 30天
  - 30天后转冷存储或删除
```

### 6.2 数据聚合

```
定时任务 (每小时):
  - 机器人健康评分 = f(在线率, 电量, 错误数)
  - 集群负载率 = 活跃任务数 / 机器人数
  - 异常检测: 传感器数据突变告警

用户报告 (每日):
  - 机器人活动摘要
  - 对话统计
  - 感知事件汇总
```

## 7. 协议优化（小改进）

### 7.1 指令预测

```
场景: 主脑连续下发运动指令, 每帧都要ACK, 延迟累积

优化: 连续同类指令无需ACK

  运动控制指令设置 NO_ACK 标志:
    - 主脑连续下发 MOVE 指令 (10Hz)
    - 机器人收到后立即执行, 不发ACK
    - 主脑通过心跳确认机器人还在

  只有以下指令必须ACK:
    - 注册、OTA、配置变更
    - 任务分配/取消
    - 紧急停止
```

### 7.2 差量心跳

```
当前: 每次心跳发送完整状态 (7字节)

优化: 差量编码

  首次心跳: 完整状态
  后续心跳: 只发变化量
    - 如果状态无变化: [短ID(2B), 0xFF(无变化标志)] → 3字节
    - 如果只有电量变化: [短ID(2B), 电量(1B)] → 3字节
    - 如果全变: 完整包 → 7字节

  节省: 典型场景节省 60% 心跳带宽
```

## 8. 安全补充

### 8.1 固件安全启动链

```
当前方案只提了签名验证, 完整的安全启动链:

  BootROM (ROM, 不可改)
    └─ 验证 bootloader 签名
        └─ 验证 app 分区签名
            └─ app 启动后验证 OTA 分区

  eFuse 配置:
    SECURE_BOOT_EN = 1
    ABS_DONE_0 = 1 (防回滚)
    JTAG_DISABLE = 1 (量产时)
```

### 8.2 隐私保护

```
摄像头/麦克风数据敏感:

  1. 机器人本地处理优先, 原始数据不上传
  2. 上传的数据端到端加密 (主脑也看不到明文)
  3. 用户可通过物理开关关闭摄像头/麦克风
     → GPIO控制摄像头电源 + LED指示灯
  4. 数据保留策略: 用户可请求删除所有数据
  5. 音频上传时自动去除背景人声 (仅保留唤醒者)
```

```
P0 (必须做):
  ├── 电源状态机 + 低功耗模式
  ├── 离线行为包
  ├── 日志系统
  └── 差量心跳

P1 (应该做):
  ├── 航位推算定位
  ├── BLE位置广播 (集群避碰)
  ├── 远程诊断
  ├── 数据生命周期
  └── 自主充电 (红外信标+导向槽)

P2 (可以后做):
  ├── 模拟器
  ├── 编队控制
  ├── 视觉里程计
  └── 隐私保护开关

P3 (Phase 2+):
  ├── SLAM
  └── 数据聚合报表
```
