# 硬件形态与演进路线

## 1. 初代目标形态

```
目标: 30-60cm 高, 一手可托举的桌面/随身陪伴机器人

基本参数:
  高度:       30-60cm
  重量:       < 800g (含电池)
  底盘直径:   12-16cm
  续航:       2-4小时 (活跃), 8小时 (待机)
  充电:       USB-C 或 桌面充电座
```

## 2. 初代结构设计

### 2.1 分层结构 (自下而上)

### 2.2 外形方案

```
方案A: 圆柱形 (推荐)
  ┌───┐
  │ ○ │  ← 头部: 圆形表情屏 + LED环
  │   │  ← 躯干: 圆柱主体
  │   │
  └─┬─┘  ← 底盘: 圆形底盘
   ═══
  优点: 结构简单, 3D打印友好, 不易倾倒
  尺寸: 直径12cm, 总高35cm

方案B: 人形简版
    ◯     ← 头部: 球形
   /|\    ← 躯干: 椭圆
   / \    ← 底盘: 双轮平衡
  优点: 拟人感强
  缺点: 结构复杂, 成本高

方案C: 蛋形
   ___
  /   \
  |   |   ← 一体成型
  \___/
  优点: 可爱, 手感好
  缺点: 内部空间利用率低
```

### 4.3 机器人底部充电触点

```
机器人底盘底部:
  ┌─────────────┐
  │    ○   ○    │  ← 2个弹簧针触点 (间距10mm)
  │             │
  │   [轮] [轮] │
  │             │
  │   [轮] [轮] │
  └─────────────┘

触点规格:
  - 弹簧针: P50-P2 弹簧针
  - 行程: 2mm
  - 额定电流: 2A
  - 接触电阻: <50mΩ
  - 安装: 焊接在底盘PCB上, 露出底盘底面
```

### 4.4 寻路充电流程

```
触发条件:
  - 电量 < 20% (警告)
  - 电量 < 10% (紧急)
  - 主脑下发充电指令
  - 空闲超过30min + 电量<50%

寻路过程:

  Phase 1: 红外信标引导 (初代)
  ┌──────────────────────────────────────────────────────────────┐
  │ 1. 机器人检测到低电量                                        │
  │ 2. 上报主脑: TASK_STATUS(充电请求)                           │
  │ 3. 主脑确认 → 下发 TASK_ASSIGN(回充电站)                     │
  │ 4. 机器人头部云台扫描 (pan 0-180°)                          │
  │    → 3个红外接收头检测信标方向                                │
  │ 5. 确定信标方位 → 转向朝向充电站                              │
  │ 6. 前进, ToF保持前方安全距离                                 │
  │ 7. 检测到充电站导向槽 (ToF距离突变)                          │
  │ 8. 减速, 沿导向槽滑入                                        │
  │ 9. 底部触点接触 → 充电电路导通                               │
  │ 10. 上报: TASK_STATUS(充电中)                                │
  │ 11. 电量>80% → 上报 TASK_STATUS(充电完成) → 离开充电站      │
  └──────────────────────────────────────────────────────────────┘

  Phase 2: 视觉辅助 (二代)
  - 摄像头识别充电站标记 (特定图案/颜色)
  - 视觉伺服对准, 提高对接成功率

  Phase 3: SLAM导航 (三代)
  - 记住充电站坐标
  - 路径规划直达
  - 不依赖红外信标
```

### 4.5 红外信标协议

```
信标调制: 38kHz载波, 脉冲位置调制 (PPM)

信号结构:
  [前导码 4ms] [充电站ID 8bit] [校验 4bit]

3个信标发射不同相位的信号:
  信标A: 相位 0°
  信标B: 相位 120°
  信标C: 相位 240°

机器人端接收:
  - 3个红外接收头 (朝前扇形分布)
  - 测量各接收头收到的信号强度(RSSI)
  - 三角定位: 计算充电站方位角

定位精度: ±15° (够用于朝向引导)
有效距离: 2-3m
```

## 5. 感知能力适配小体型

### 4.1 视觉 (减配适配)

```
初代方案:
  摄像头: OV2640 (已足够小, 无需改动)
  深度: 暂不装深度相机 (体积/功耗限制)
       替代: 3× ToF VL53L0X (量程2m, 足够室内避障)
  本地推理: ESP32-S3 NPU 做轻量人脸检测

演进:
  V2: 增加VL53L5CX (8×8区域深度, 小型模组)
  V3: 增加OV5640 (500万像素) + 更强NPU
```

### 4.2 音频 (保持完整版)

```
初代方案:
  麦克风: 4× INMP441 (I2S, 体积小, 适合小机器人)
  阵列间距: 3-4cm (受限于头部尺寸)
  波束成形: 简单延迟求和 (间距小, 精度够用)
  唤醒词: ESP-SR (本地, 低功耗)
  扬声器: 3W微型 (音量够室内使用)
```

### 4.3 触觉 (简化)

```
初代方案:
  触摸区域: 躯干外壳的电容触摸区域 (2-4个)
  实现: PCB上的铜箔走线, 覆盖导电布
  不用MPR121阵列, 直接用ESP32-S3内置触摸引脚
  检测: 轻触/长触/滑动

演进:
  V2: 增加FSR压力传感器 (拥抱检测)
  V3: 全身触觉皮肤
```

### 4.4 环境 (保持)

```
初代方案:
  BME280 + BH1750 (小封装, I2C, 不占空间)
  SGP30 空气质量 (可选, 增加~10元成本)
```

## 5. 演进路线

```
Phase 1: MiniBot (30-40cm)          ← 初代目标
├── 全向底盘 + 头部云台
├── 视觉: 摄像头 + ToF避障
├── 音频: 4ch麦克风阵列 + 扬声器
├── 环境: 温湿度/光照
├── 触觉: 电容触摸
├── 续航: 2小时
└── BOM: ~380 RMB

Phase 2: CompanionBot (50-60cm)     ← 二代升级
├── 增加深度传感器 (VL53L5CX)
├── 增加手臂 (2自由度 × 2, 简单抓取)
├── 增加底部升降 (高度可调 50-70cm)
├── 音频: 6ch麦克风阵列 (声源定位更准)
├── 触觉: FSR压力 + 电容触摸
├── 电池: 5000mAh, 续航4小时
├── 可选: 4G模块 (户外场景)
└── BOM: ~800 RMB

Phase 3: HomeBot (80-100cm)         ← 三代
├── 深度相机 (RealSense D435i)
├── 激光雷达 (RPLIDAR A1, 2D建图)
├── 机械臂 (4自由度 × 2, 简单操作)
├── 底盘: 差速轮 + 万向轮 (承重增加)
├── 音频: 8ch环形阵列
├── 全身触觉皮肤
├── 电池: 10000mAh, 续航6小时
├── 协处理器: K210/NVIDIA Jetson Nano
└── BOM: ~2000 RMB

Phase 4: ProBot (120cm+)            ← 四代
├── 6自由度双臂 (承重5kg)
├── 深度相机 + 3D激光雷达
├── 4轮独立悬挂底盘
├── GPU板载推理
├── 电池: 20000mAh, 续航8小时
└── BOM: ~5000 RMB
```

### 5.1 架构兼容性

```
所有阶段共享:
  ├── 同一套BCP通信协议 (向下兼容)
  ├── 同一套主脑服务 (能力集自动适配)
  ├── 同一套FreeRTOS固件框架 (模块化裁剪)
  └── 同一套集群管理 (能力集注册)

差异处理:
  - 机器人注册时上报能力集
  - 主脑按能力集调度 (不给MiniBot分配需要机械臂的任务)
  - 固件通过 feature flag 裁剪 (编译时) 或 动态检测 (运行时)

固件模块化:
  firmware/
  ├── core/           — 通信层 + 系统层 (所有阶段必须)
  ├── perception/     — 感知层
  │   ├── camera/     — Phase 1+
  │   ├── audio/      — Phase 1+
  │   ├── depth/      — Phase 2+
  │   ├── lidar/      — Phase 3+
  │   └── touch/      — Phase 1+(简化) Phase 2+(完整)
  ├── motion/         — 运动层
  │   ├── wheels/     — Phase 1+
  │   ├── arms/       — Phase 2+
  │   └── lift/       — Phase 2+
  └── expression/     — 表达层
      ├── led/        — Phase 1+
      ├── screen/     — Phase 1+
      └── voice/      — Phase 1+
```
