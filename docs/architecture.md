# Companion Robot 集群系统 — 总体架构设计

## 1. 系统概述

本系统是一个基于 ESP32-S3 机器人终端 + Rust 主脑服务的分布式陪伴机器人集群。
支持 50+ 机器人并发调度，具备完整的视觉、音频、环境、触觉感知能力。

### 初代形态

初代机器人高度 30-40cm，重量 < 800g，一手可托举。采用圆柱形设计，底部麦克纳姆轮全向移动，
头部2自由度云台 + OLED表情屏。详见 [hardware-form-factor.md](hardware-form-factor.md)。

### 演进路线

架构支持从 30cm MiniBot 到 120cm+ ProBot 的平滑演进：
- 通信协议(BCP)向下兼容，新指令不影响旧固件
- 能力集注册机制，主脑自动适配不同阶段的机器人
- 固件模块化，通过 feature flag 裁剪

```
Phase 1 MiniBot (30-40cm)  →  Phase 2 CompanionBot (50-60cm)
  全向底盘+头部云台              增加深度传感器+手臂
  摄像头+ToF避障                 6ch麦克风+FSR触觉
  4ch麦克风+电容触摸             电池升级5000mAh

Phase 3 HomeBot (80-100cm) →  Phase 4 ProBot (120cm+)
  深度相机+激光雷达              6自由度双臂+3D激光
  机械臂+全身触觉                GPU板载推理
  Jetson Nano协处理器            20000mAh长续航
```

## 3. 三层架构模型

### 3.1 感知层 (Perception Layer) — 机器人本地

| 子系统 | 传感器 | 处理方式 | 延迟要求 |
|--------|--------|----------|----------|
| 视觉 | OV2640摄像头 + VL53L0X×3 ToF | 本地运动检测，复杂识别上传主脑 | <100ms |
| 音频 | INMP441 I2S麦克风阵列(4ch) | 本地VAD+唤醒词，ASR上传 | <50ms(VAD) |
| 环境 | BME280(温湿度气压) + BH1750(光照) + SGP30(空气质量) | 本地采样，周期上报 | 1s |
| 姿态 | BMI270 IMU | 本地融合滤波 | <10ms |
| 触觉 | ESP32-S3内置电容触摸(Phase1) / MPR121阵列(Phase2+) | 本地中断检测 | <5ms |
| 避障 | VL53L0X ToF × 3 | 本地实时 | <20ms |

### 3.2 执行层 (Execution Layer) — 机器人本地

- **运动控制**: DRV8833双H桥驱动 × 2 → N20电机 × 4 (麦克纳姆轮全向移动)，本地PID闭环
- **头部云台**: SG90微型舵机 × 2 (pan + tilt)
- **LED/灯光**: WS2812B灯环 × 8，情绪表达
- **音频输出**: MAX98357A I2S DAC + 3W微型喇叭，TTS播报
- **表情屏**: SSD1306 OLED 1.3" 128×64，表情动画

### 3.3 决策层 (Decision Layer) — 主脑

- **任务调度**: 基于优先级的多机器人任务分配
- **路径规划**: 多机协同避障（集群场景）
- **对话管理**: 上下文感知的多轮对话
- **行为引擎**: 状态机驱动的行为树

## 4. 通信模式

### 4.1 直连模式 (Direct Mode)

```
机器人 ◄──WebSocket/TCP──► 主脑网关
```

- 适用: 内网、稳定WiFi环境
- 协议: 自定义二进制帧 over WebSocket
- 延迟: <10ms (局域网)
- 特点: 主脑完全控制，指令直达

### 4.2 桥接模式 (Bridge Mode)

```
机器人 ◄──BLE──► 移动设备 ◄──WebSocket──► 主脑
```

- 适用: 机器人无外网、移动设备作为中继
- 协议: BLE GATT自定义服务 + 移动端SDK
- 延迟: <50ms (BLE) + 网络延迟
- 特点: 移动设备做协议转换和缓存

### 4.3 转发模式 (Relay Mode)

```
机器人 ◄──BLE──► 移动设备 ──► 第三方AI API
```

- 适用: 主脑离线、需要大模型推理
- 协议: 移动设备封装API调用
- 延迟: 取决于第三方API
- 特点: 降级模式，功能受限

### 4.4 模式切换策略

```rust
// 机器人端连接管理器伪代码
enum ConnMode { Direct, Bridge, Relay }

struct ConnManager {
    mode: ConnMode,
    direct: Option<WebSocket>,
    bridge: Option<BleConn>,
    fallback_timer: Instant,
}

impl ConnManager {
    // 优先级: Direct > Bridge > Relay
    // 切换条件:
    //   Direct不可达 5s → 切Bridge
    //   Bridge不可达 10s → 切Relay
    //   Direct恢复 → 立即切回
    fn auto_switch(&mut self) { ... }
}
```

## 5. 集群管理

### 5.1 机器人注册流程

```
1. 机器人上电 → 读取eFuse中的机器码(128bit UUID)
2. 广播发现(mDNS/UDP) 或 连接预配置的主脑地址
3. 发送注册包: [机器码 + 能力集 + 固件版本]
4. 主脑验证机器码签名 → 分配集群内短ID(16bit)
5. 机器人收到短ID后，后续通信使用短ID节省带宽
```

### 5.2 能力集描述

```rust
struct RobotCapabilities {
    has_camera: bool,          // 视觉
    has_depth_sensor: bool,    // 深度
    has_mic_array: bool,       // 麦克风阵列
    has_speaker: bool,         // 扬声器
    has_imu: bool,             // 姿态
    has_touch: bool,           // 触觉
    has_tof_x3: bool,          // ToF避障
    motor_count: u8,           // 电机数量
    led_count: u16,            // LED数量
    has_screen: bool,          // 表情屏
    max_speed_cm_s: u16,       // 最大速度
    battery_capacity_mah: u16, // 电池容量
}
```

### 5.3 心跳与状态上报

```
心跳间隔: 5s (可配置)
心跳包: [短ID(2B) + 状态标志(1B) + 电池%(1B) + 信号强度(1B) + 当前任务ID(2B)]

状态标志位:
  bit0: 运动中
  bit1: 充电中
  bit2: 低电量警告
  bit3: 传感器故障
  bit4: 通信异常
  bit5-7: 保留

超时判定: 3次心跳未收到 → 标记离线 → 触发告警
```

### 5.4 集群调度策略

```
任务队列 → 调度器 → 分配给最优机器人

调度因子:
  - 机器人位置(距离目标最近)
  - 当前负载(空闲优先)
  - 电量(>20%优先)
  - 能力匹配(任务需要摄像头 → 有摄像头的机器人)
  - 亲和性(同一用户尽量分配同一机器人)

调度算法: 加权评分 + 软亲和性
  score = w1*distance + w2*load + w3*battery + w4*capability_match + w5*affinity
```

## 6. 数据流概览

```
传感器数据流 (机器人 → 主脑):
  视觉帧 → 本地运动检测 → 有运动? → JPEG压缩 → WebSocket → 主脑AI推理
  音频流 → VAD检测 → 有语音? → PCM/Opus → WebSocket → 主脑ASR
  环境数据 → 1s采样 → 打包 → 心跳附带上报
  触觉事件 → 中断触发 → 立即上报

控制指令流 (主脑 → 机器人):
  任务指令 → 调度器 → 集束编码 → WebSocket → 机器人解码 → 执行
  紧急指令 → 绕过队列 → 最高优先级直达
```

## 7. 技术栈总结

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 机器人MCU | ESP32-S3-WROOM-1 N16R8 | 双核240MHz, 512KB SRAM + 8MB PSRAM, WiFi+BLE, NPU |
| 机器人OS | ESP-IDF (FreeRTOS) | 官方SDK，实时调度 |
| 视觉推理 | ESP32-S3 NPU (Phase1) / K210 (Phase2+) | 轻量推理卸载 |
| 主脑语言 | Rust | 内存安全，高性能并发 |
| 通信网关 | tokio-tungstenite | 异步WebSocket |
| 消息队列 | NATS JetStream | 轻量、高性能、持久化 |
| 数据库 | PostgreSQL + TimescaleDB | 关系数据 + 时序传感器数据 |
| 缓存 | Redis | 设备状态、会话缓存 |
| 容器编排 | Docker Compose (小规模) / K8s (大规模) | 服务部署 |
