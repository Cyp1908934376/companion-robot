# 性能基准与SLA

## 1. SLA定义

### 1.1 可用性

```
目标: 99.9% (每月停机 < 43分钟)

度量:
  可用性 = 成功请求数 / 总请求数 × 100%

排除:
  - 计划维护窗口 (每月2小时, 凌晨)
  - 不可抗力 (网络运营商故障)
  - 机器人端硬件故障 (不计入SLA)
```

### 1.2 延迟SLA

```
┌─────────────────────────┬────────┬────────┬────────┐
│ 指标                    │ p50    │ p95    │ p99    │
├─────────────────────────┼────────┼────────┼────────┤
│ 指令下发延迟 (WiFi直连) │ 10ms   │ 30ms   │ 80ms   │
│ 指令下发延迟 (BLE桥接)  │ 50ms   │ 120ms  │ 250ms  │
│ 心跳往返                │ 5ms    │ 15ms   │ 40ms   │
│ 任务分配延迟            │ 50ms   │ 150ms  │ 300ms  │
```

## 2. 端到端延迟预算

```
场景: 主脑下发运动指令, WiFi直连

  ┌──────────────────────────────────────────────────────────┐
  │  主脑应用层         │  5ms   │  调度决策 + BCP编码        │
  │  NATS发布           │  1ms   │  消息队列                  │
  │  Gateway处理        │  2ms   │  BCP编码 + TCP发送         │
  │  网络传输           │  2ms   │  局域网WiFi                │
  │  ESP32接收+解码     │  1ms   │  BCP解析                   │
  │  指令分发           │  0.5ms │  FreeRTOS队列              │
  │  电机PID控制        │  0.5ms │  PWM更新                   │
  │  ─────────────────────────────────────────────────────  │
  │  总计               │  12ms  │  远低于100ms目标           │
  └──────────────────────────────────────────────────────────┘

场景: 语音对话, BLE桥接

  ┌──────────────────────────────────────────────────────────┐
  │  机器人录音          │  200ms │  VAD检测 + 缓冲           │
  │  BLE传输             │  50ms  │  音频流分片               │
  │  手机ASR             │  500ms │  本地语音识别             │
  │  手机NLU             │  100ms │  意图识别                 │
  │  手机TTS             │  200ms │  语音合成                 │
  │  BLE回传             │  50ms  │  音频流分片               │
  │  机器人播放          │  50ms  │  I2S输出                  │
  │  ─────────────────────────────────────────────────────  │
  │  总计               │  1150ms│  用户可接受               │
  └──────────────────────────────────────────────────────────┘
```

## 3. 容量规划

### 3.1 单机器人资源消耗

```
带宽 (上行, 机器人→主脑):
  心跳:         7B × 0.2Hz = 1.4 B/s
  传感器数据:   50B × 1Hz = 50 B/s (聚合后)
  音频流:       8KB/s (Opus 64kbps, 说话时)
  图像快照:     20KB × 0.1Hz = 2 KB/s (运动触发时)
  事件:         ~100 B/s (偶发)
  ────────────────────────────────
  平均:         ~53 B/s (安静)
  峰值:         ~10 KB/s (说话+拍照)

带宽 (下行, 主脑→机器人):
  控制指令:     20B × 10Hz = 200 B/s (运动时)
  LED/表情:     10B × 1Hz = 10 B/s
  TTS音频:      8KB/s (回复时)
  任务分配:     ~50 B/s (偶发)
  ────────────────────────────────
  平均:         ~210 B/s
  峰值:         ~8.2 KB/s

连接数:
  WebSocket:    1长连接/机器人
  NATS订阅:     3个/机器人 (cmd, task, config)
```

### 3.2 集群资源规划

```
50台机器人集群:

  Gateway (tokio-tungstenite):
    连接数: 50
    CPU: ~5% (单核)
    内存: ~100KB (50 × 2KB)
    带宽: 上行 ~500KB/s, 下行 ~500KB/s
    实例: 1个足够

  Device Manager:
    CPU: ~2% (主要是心跳处理)
    内存: ~50MB (状态缓存)
    实例: 1个

  Scheduler:
    CPU: ~5% (任务分配计算)
    内存: ~20MB (任务队列)
    实例: 1个 (主备)

  AI推理 (假设50%机器人同时对话):
    并发: 25请求
    GPU: 1× NVIDIA T4 (Whisper + YOLO)
    显存: ~4GB
    CPU: ~4核
    实例: 1个

  NATS:
    消息速率: ~500 msg/s
    存储: ~100MB/h (JetStream)
    内存: ~512MB
    实例: 1个 (生产建议3节点)

  PostgreSQL:
    连接数: ~20
    存储: ~1GB/月 (传感器数据)
    内存: ~512MB
    实例: 1个

  Redis:
    连接数: ~100
    内存: ~50MB (设备状态)
    实例: 1个

总计:
  CPU: ~8核
  内存: ~4GB
  GPU: 1× T4
  存储: ~50GB
  带宽: ~1Mbps (局域网)
```

### 3.3 扩展公式

```
Gateway实例数 = ceil(机器人数 / 1000)
AI推理实例数 = ceil(并发对话数 / 50)
NATS节点数 = 机器人数 > 200 ? 3 : 1
PostgreSQL = 主从 (机器人数 > 100)
Redis = Sentinel (机器人数 > 50)

带宽需求:
  上行 = 机器人数 × 10KB/s (峰值)
  下行 = 机器人数 × 8.2KB/s (峰值)
  50台: ~900KB/s, 100台: ~1.8MB/s, 500台: ~9MB/s
```

## 4. 资源使用基准

### 4.1 正常负载

```
Gateway (50连接, 活跃):
  CPU: 3-5%
  内存: 80-120MB
  网络: 50KB/s 上下行
  连接数: 50

Scheduler (50任务/小时):
  CPU: 1-2%
  内存: 15-25MB
  NATS发布: 10 msg/s

AI服务 (10并发推理):
  GPU: 30-40%
  显存: 2GB
  CPU: 2核
  推理延迟: ASR 400ms, 视觉 80ms
```

### 4.2 峰值负载

```
Gateway (1000连接, 50%活跃):
  CPU: 30-40%
  内存: 2GB
  网络: 5MB/s 上下行
  消息处理: 3000 msg/s

Scheduler (500任务/小时):
  CPU: 10-15%
  内存: 50MB
  NATS发布: 100 msg/s

AI服务 (50并发推理):
  GPU: 80-90%
  显存: 6GB
  CPU: 8核
  推理延迟: ASR 800ms, 视觉 150ms
```

## 5. 降级行为

```
┌─────────────────────┬──────────────────────────────────────────┐
│ 触发条件            │ 降级行为                                 │
├─────────────────────┼──────────────────────────────────────────┤
│ Gateway CPU > 80%   │ 拒绝新连接, 返回E_CONN_REJECTED          │
│                     │ 已有连接正常服务                         │
├─────────────────────┼──────────────────────────────────────────┤
│ Gateway内存 > 90%   │ 清理断开连接的会话缓存                   │
│                     │ 拒绝新连接                               │
├─────────────────────┼──────────────────────────────────────────┤
│ NATS消息积压 > 10K  │ 丢弃低优先级传感器数据                   │
│                     │ 保留控制指令和心跳                       │
├─────────────────────┼──────────────────────────────────────────┤
│ AI服务GPU > 90%     │ 熔断器触发, 降级为本地处理               │
│                     │ 排队超过10s的请求直接拒绝                │
├─────────────────────┼──────────────────────────────────────────┤
│ PostgreSQL连接池满  │ 读请求走Redis缓存                       │
│                     │ 写请求排队 (最多100)                     │
├─────────────────────┼──────────────────────────────────────────┤
│ Redis内存满         │ LRU淘汰低频设备状态                     │
│                     │ 心跳重新写入                             │
├─────────────────────┼──────────────────────────────────────────┤
│ 网络带宽 > 80%      │ 降低图像传输频率 (0.1Hz→0.02Hz)         │
│                     │ 音频降级为低码率 (64k→32k)              │
└─────────────────────┴──────────────────────────────────────────┘
```

## 6. 监控指标

```
延迟指标:
  gateway_cmd_latency_ms          — 指令转发延迟
  scheduler_assign_latency_ms     — 任务分配延迟
  ai_inference_latency_ms         — AI推理延迟
  e2e_command_latency_ms          — 端到端指令延迟

吞吐量指标:
  gateway_msgs_per_second         — 网关消息速率
  nats_publish_rate               — NATS发布速率
  nats_consume_rate               — NATS消费速率
  ai_requests_per_second          — AI请求速率

资源指标:
  gateway_cpu_usage               — 网关CPU
  gateway_memory_bytes            — 网关内存
  gateway_connections              — 网关连接数
  scheduler_queue_depth           — 调度队列深度
  gpu_utilization                 — GPU使用率
  db_connection_pool_usage        — 数据库连接池

业务指标:
  robot_online_count              — 在线机器人数
  task_success_rate               — 任务成功率
  task_timeout_rate               — 任务超时率
  heartbeat_miss_rate             — 心跳丢失率
  sensor_data_gap_rate            — 传感器数据间隙率
```
