# 安全设计

## 1. 机器码与身份认证

### 1.1 机器码生成

```
烧录阶段 (工厂):
  1. 生成 128bit UUID (随机)
  2. 写入 eFuse (一次性可编程, 不可读出)
  3. 用根私钥对 UUID 签名 → 证书
  4. 证书写入 NVS 只读分区

运行时:
  - 机器码: eFuse中的UUID (硬件级, 不可提取)
  - 身份证明: NVS中的证书 (可出示给主脑验证)
  - 挑战-响应: 主脑发随机数 → 机器人用eFuse密钥签名 → 验证
```

### 1.2 注册认证流程

```
机器人                                    主脑
  │                                         │
  │──── CONNECT (WebSocket/TCP) ───────────►│
  │                                         │
  │◄──── CHALLENGE (随机数 32B) ────────────│
  │                                         │
  │──── RESPONSE (签名 64B, 证书NB) ──────►│
  │                                         │
  │                      验证证书链 ────────┤
  │                      验证签名 ──────────┤
  │                      检查黑名单 ────────┤
  │                                         │
  │◄──── AUTH_OK (短ID, 会话密钥) ─────────│
  │                                         │
  │──── REGISTER (能力集, 固件版本) ──────►│
  │                                         │
  │◄──── REG_ACK (配置参数) ───────────────│
  │                                         │
  │         后续通信使用会话密钥加密          │
```

### 1.3 证书格式

```
Robot Certificate:
  version:        1
  serial:         机器码 UUID
  issuer:         根CA
  validity:       10年
  public_key:     Ed25519公钥
  extensions:
    capabilities: 硬件能力集 (工厂写入)
    batch:        生产批次
  signature:      根CA签名
```

## 2. 通信加密

### 2.1 WebSocket (外网)

```
TLS 1.3 强制
  - 证书: Let's Encrypt 或自签CA
  - 密码套件: TLS_AES_256_GCM_SHA384
  - HSTS启用
  - 证书固定 (Certificate Pinning): 机器人端硬编码CA指纹

双向认证 (mTLS):
  - 服务端证书: 主脑域名证书
  - 客户端证书: 机器人出厂证书
  - 验证: 双方互验证书链
```

### 2.2 TCP (内网)

```
会话密钥协商:
  1. 认证阶段完成后, 双方共享一个 256bit 会话密钥
  2. 会话密钥通过 ECDH (X25519) 协商
  3. 后续所有BCP帧使用 AES-256-GCM 加密

帧加密:
  明文BCP帧 → AES-256-GCM(密文, nonce, tag) → 传输

  nonce = 帧序列号(SeqNo) + 会话IV (12字节)
  tag   = 16字节认证标签
  附加数据(AAD) = 帧头(前8字节, 不含CRC)

  加密后帧:
  ┌────────────────────────────────────────┐
  │ 帧头(8B, 明文) │ 密文(NB) │ GCM Tag(16B) │
  └────────────────────────────────────────┘
  CRC基于整个帧计算(含密文和tag)
```

### 2.3 BLE

```
BLE Security Level 4:
  - LE Secure Connections (LESC)
  - 配对方式: 数字比较 (Numeric Comparison)
  - 加密: AES-CCM 128bit
  - 密钥生成: ECDH P-256

首次配对流程:
  1. 机器人进入配对模式 (按键触发)
  2. 移动设备扫描发现
  3. LESC配对, 用户确认数字
  4. 长期密钥存储 (LTK)
  5. 后续自动重连

安全限制:
  - 配对超时 60秒
  - 连续3次配对失败 → 锁定 5分钟
  - 未知设备连接 → 拒绝
```

## 3. 权限模型

### 3.1 角色定义

```
角色层级:

  Admin (管理员)
    ├── 注册/注销机器人
    ├── 集群配置
    ├── OTA推送
    ├── 查看所有数据
    └── 修改权限

  Operator (操作员)
    ├── 创建/取消任务
    ├── 查看机器人状态
    ├── 查看传感器数据
    └── 紧急停止

  User (普通用户)
    ├── 分配到的机器人交互
    ├── 基础对话
    └── 查看自己的任务

  Robot (机器人身份)
    ├── 上报传感器数据
    ├── 接收指令
    └── 心跳
```

### 3.2 认证令牌

```
JWT Token (管理面REST API):

Header:
  { "alg": "EdDSA", "typ": "JWT" }

Payload:
  {
    "sub": "user_id",
    "role": "admin|operator|user",
    "robots": ["robot_id_1", "robot_id_2"],  // 可访问的机器人
    "exp": 1735689600,
    "iat": 1735603200
  }

签名: Ed25519

有效期: Admin 4h, Operator 8h, User 24h
刷新: 使用 refresh_token (7天有效)
```

### 3.3 指令权限矩阵

```
指令类型          Admin  Operator  User  Robot
─────────────────────────────────────────────
运动控制           ✓      ✓        ✗     ✗
表达控制           ✓      ✓        ✗     ✗
任务分配           ✓      ✓        ✗     ✗
OTA更新           ✓      ✗        ✗     ✗
配置修改           ✓      ✗        ✗     ✗
传感器查询         ✓      ✓        限定   ✗
心跳/注册          ✗      ✗        ✗     ✓
状态上报           ✗      ✗        ✗     ✓

限定: User只能查看分配给自己的机器人的传感器数据
```

## 4. 安全加固

### 4.1 防重放攻击

```
机制:
  - 每帧包含 SeqNo (单调递增)
  - 接收方维护滑动窗口, 拒绝过期序列号
  - 窗口大小: 256帧
  - 时间戳校验: 帧内时间与本地时间差 > 30s → 拒绝
```

### 4.2 防中间人

```
- mTLS双向认证 (WebSocket/TCP)
- LESC配对 (BLE)
- 证书固定 (Certificate Pinning)
- 会话密钥绑定设备身份
```

### 4.3 固件安全

```
- 固件签名: Ed25519, 私钥离线存储
- 安全启动: eFuse中烧录公钥哈希, 启动时验证签名
- 回滚保护: 版本号单调递增, 拒绝低版本固件
- 分区加密: AES-256-XTS 加密app分区
```

### 4.4 密钥管理

```
密钥层级:
  根密钥 (Root Key)
    ├── 硬件安全模块 (HSM) 或 离线保管
    ├── 用于签发机器人证书
    └── 从不联网

  设备密钥 (Device Key)
    ├── Ed25519 密钥对
    ├── 私钥在eFuse, 不可提取
    └── 用于认证和签名

  会话密钥 (Session Key)
    ├── ECDH协商, 每次连接重新生成
    ├── AES-256-GCM加密通信
    └── 连接断开即销毁

  API密钥 (API Key)
    ├── JWT签名密钥
    ├── 轮换周期: 90天
    └── 支持多版本共存 (滚动轮换)
```

## 5. 审计日志

```
记录事件:
  - 机器人注册/注销
  - 认证成功/失败
  - OTA更新
  - 权限变更
  - 紧急停止
  - 异常行为检测

日志格式 (JSON):
  {
    "timestamp": "2026-05-25T10:30:00Z",
    "event": "auth_success",
    "actor": "robot:uuid-xxx",
    "target": "gateway-01",
    "detail": { "short_id": 42, "method": "certificate" },
    "ip": "192.168.1.100"
  }

存储: Elasticsearch / Loki (保留 90 天)
告警: 连续认证失败 > 5次 → 封禁IP + 告警
```
